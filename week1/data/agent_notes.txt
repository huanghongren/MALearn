ReAct 是一种将推理与行动结合的 agent 框架。核心思想是让模型先思考下一步该做什么，再执行动作获取外部信息，然后根据观察结果继续推理。它适合问答、工具调用和检索增强任务。

Toolformer 的核心思想是让语言模型学会在合适的时候调用外部工具。它强调模型不仅要会回答，还要能决定何时使用搜索、计算器、翻译器或数据库等工具。

Reflexion 的核心思想是让智能体在失败后生成文字形式的反思，再把这些反思作为下一轮决策依据。它强调通过语言反馈而不是重新训练参数来提升表现。

LangGraph 的核心概念包括 State、Node 和 Edge。State 表示流程中的共享状态，Node 表示执行步骤，Edge 表示节点之间的流向。条件边允许系统根据当前状态动态选择下一跳。

多智能体系统的核心不是数量，而是分工。常见模式包括 supervisor-specialist，其中 supervisor 负责调度和汇总，specialist 负责处理具体任务，例如科研解释或代码帮助。

Reflection + Retry 机制表示系统先生成第一版答案，再通过检查器发现明显问题。如果答案存在草稿痕迹、缺少关键信息或结果错误，就生成反思意见并重试一次。
